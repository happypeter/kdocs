<key>mediaplayer</key>

<type>mp3</type>

<icon>mp3.gif</icon>

as I see it, the mine file does this job: of a file is of the type of mp3, then its icon shall be mp3.gif, and if the icon pressed, mediaplayer is actived to play the file.

5. now I have a idea of the whole qtopia sytem, I know it is not perfectlly right, but it makes sense. Every visuable is a xml page. Booting the system a program named sever runs first(sever –qws in rc.S), and show me many icons on the sreen, you press the a icon, say calculator icon, this is a signal to sever,then the exe file of calculator is activated by sever. When the calculator exe start, it will open its own xml, note, it is the exe active its xml, not xml actives its appliation exe, that is why you can not find the name of exe in the corresponding xml. The caculator is activated by the sever.

6. anyway, as soon as the caculator is activated, it starts to plays the role of a sever, when the small icons(say the number buttons) are pressed, the exe will react and sent the result back to the xml(the screen of the calcalator), so the xml is a real ASP, it commubicate with its exe very well.
